## 三、竞品商业模式分析

### 3.1 CodeRabbit

```
模式：SaaS订阅制
├── 免费层（引流）
│   ├── PR摘要
│   ├── IDE内审查
│   └── 公共仓库免费
├── Pro层（主要营收）
│   ├── 无限PR审查
│   ├── 工具集成
│   └── 分析仪表板
└── Enterprise层（高价值客户）
    ├── 私有化部署
    ├── SLA支持
    └── API访问
```

### 3.2 PR-Agent

```
模式：开源 + 商业化（Qodo）
├── 开源版（社区运营）
│   ├── 完整核心功能
│   ├── 自托管
│   └── 需自行承担LLM成本
└── 商业版（Qodo服务）
    ├── 托管服务
    ├── 技术支持
    └── 企业级功能
```

### 3.3 Sourcery

```
模式：SaaS + 企业级
├── 免费试用（引流）
├── Pro版（主要营收）
│   ├── 完整审查功能
│   └── 安全扫描
└── Enterprise版（高价值客户）
    ├── BYO LLM
    ├── SOC 2合规
    └── 定制化服务
```

### 5.2 差异化竞争功能

| 功能 | 优先级 | 差异化价值 |
| --- | :---: | --- |
| **Gitee集成** | P0 | 国产平台唯一支持 |

### 5.3 创新功能建议

| 功能 | 创新点 | 说明 |
| --- | --- | --- |

## 六、技术实现建议

### 6.1 GitHub Bot实现参考

```
GitHub Bot架构：
├── GitHub App注册
│   ├── Webhook接收PR事件
│   ├── 获取PR diff内容
│   └── 以Bot身份发布评论
├── 核心审查引擎
│   ├── 代码解析
│   ├── LLM分析
│   └── 结果格式化
└── 命令处理器
    ├── /review - 触发审查
    ├── /improve - 改进建议
    ├── /explain - 代码解释
    └── /summary - PR摘要
```

## 七、总结

### 7.1 竞品格局

| 竞品 | 定位 | 优势 | 劣势 |
| --- | --- | --- | --- |
| CodeRabbit | 商业化领先者 | 功能全面、用户量大 | 价格高、闭源 |
| PR-Agent | 开源领先者 | 完全开源、可自托管 | 部署复杂、维护成本 |
| Sourcery | 安全聚焦者 | 安全能力强、合规认证 | 平台支持少 |

### 7.2 我们的机会

**差异化定位建议**：国产化 + 开源 + 低价

1. **国产化**：首个支持Gitee/Coding的AICR产品
2. **开源**：提供MIT/Apache许可的开源版本
3. **低价**：比CodeRabbit更有竞争力的定价
4. **中文优化**：针对中文开发者特别优化
5. **国产LLM**：支持国产大模型，满足合规需求
