## 二、产品定位

Sourcery的核心理念是：**"AI加速了编码，但也加速了Bug、漏洞和技术债务。传统代码审查无法跟上，这就是Sourcery的价值。"**

主要解决的问题：
- AI辅助编码导致代码量激增
- 传统Code Review跟不上开发速度
- Bug、安全漏洞和技术债务快速累积

## 四、GitHub集成方式

### 4.1 PR阶段集成

- 对GitHub上每一次代码变更进行即时审查
- 自动在PR中提供代码总结和修复建议
- 以Bot身份在PR评论区发布审查结果

### 4.2 跨仓库扫描

- 连接并扫描所有GitHub仓库
- 持续进行安全分析和代码质量检查

### 4.3 多平台支持

| 平台 | 支持情况 |
| --- | --- |
| GitHub | ✓ 完整支持 |
| GitLab | ✓ 完整支持 |
| VS Code | ✓ 插件支持 |
| PyCharm | ✓ 插件支持 |

## 五、技术特点

### 5.1 企业级安全与隐私

| 特点 | 说明 |
| --- | --- |
| **不使用代码训练** | 确保代码知识产权安全 |
| **零数据保留** | 与可信提供商合作，提供零保留选项 |
| **BYO LLM** | 支持企业自带大模型端点（Bring Your Own LLM） |
| **SOC 2认证** | 符合严格的安全标准 |

### 5.2 性能优化

| 特点 | 说明 |
| --- | --- |
| AI规模设计 | 专为处理AI产生的大量代码设计 |
| 高效率 | 缩短审查周期，减少阻碍 |
| 快速合并 | 加快代码合并速度 |

## 八、竞争优势分析

### 8.1 优势

| 优势 | 说明 |
| --- | --- |
| **安全聚焦** | 强调安全扫描，安全漏洞检测能力强 |
| **隐私保护** | 不使用代码训练，零数据保留 |
| **BYO LLM** | 支持自带模型，满足企业合规需求 |
| **SOC 2认证** | 企业级安全合规认证 |
| **IDE深度集成** | VS Code/PyCharm原生支持 |
| **AI代理集成** | 可与其他AI编码工具协同工作 |

### 8.2 劣势

| 劣势 | 说明 |
| --- | --- |
| **平台支持有限** | 仅支持GitHub/GitLab，不支持Bitbucket等 |
| **IDE限制** | 主要支持VS Code和PyCharm |
| **定价不透明** | 官网定价信息不够清晰 |
| **社区规模** | 相比CodeRabbit知名度稍低 |
